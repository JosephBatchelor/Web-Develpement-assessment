CSS positioned layout

To start off with this weeks lecture we will be looking into position properties. The main purpose of posistioned layouts is to remove elements and have the ability to place them anywhere on your webpage. This can be done by simply using some styling lines and values. Typically, without the thought or use of positioned layout types. Your webpage should be displayed within a normal flow layout, this means that inline elements are nested within block elements. However, CSS has allowed us to use different layouts which affects the elements within a HTML document. Listed below are some of the following CSS lines that can be used.

  • Static – This is where elements are positioned in a normal flow layout.
  • Relative - This removes the elements normal flow layout and allows you to place it anywhere on your page.
  • Sticky – This holds an element in place which can allow users to scroll whilst having the element remain in place.
  • Absolute – Positions an element within a parent element.
  • Fixed – Almost the same as absolute, this removes the element and positions it in relation to the viewport.

Common uses

Now we will talk about the use of fixed position and sticky layout to give an understanding of how and where these can be used. These two layouts are commonly used and are often compared to one another, therefore it is necessary to explain their differences and uses.

Although it is a relatively new feature to CSS, sticky layout has become increasingly popular. The purpose of this layout is to stick elements in place allowing the user to move freely around the webpage whilst maintaining the ability to still see that element. I have applied this feature to my navigation bar, meaning that when a user scrolls through my page the navigation bar follows them. Resulting in quick access to other pages or sections within the page itself. This involves the use of scripts in-order for this to work however, this is a feature used in many other webpages to improve user experience.



Now to discuss fixed layout, this layout feature is used as default normal flow. Everything remains in place. An example of this being used within my learning journal has mainly been in the weekly content. All the information about lectures are within a fixed layout which means they remain still when scrolling through my webpage. Overall this means that the presentation will remain the same compared to that of fluid or flexible layouts.

Role of forms on websites

Within this section we will be introducing web forms. A web form is essentially a section on a webpage or an independent webpage that allows users to interact, utilising the web form input as a means of obtaining data. Web forms have been largely used with commercial sites to obtain information from customers.

A web form is made up of three components; the front-end, the middleware and the back end. The front-end is where clients can access the form on a device, the front-end transfers the input data that the customer may have inserted which then travels through the middleware which links to software such as databases. Once travelled, the data is then placed in the back-end which is the data base / server where the data is held for later use.

We will only be making the front-end of the web form this semester, we will be starting the back-end in semester 2.

Introduction to responsive web design/development

Responsive web design is the process of designing websites/webpages that can render when used on a variety of devices. This means that the size of the screen is able to change when viewed on different platforms.

RWD was designed by Ethan Marcotte and is used widely by the majority, if not all, web developers. It is useful because people can access websites from a variety of different devices, RWD helps developers to scale their site to the ideal size for different platforms. Another aspect is cost effectiveness, it is cheaper for most businesses as they don’t have to create a second webpage for a different device. The main goal when using RWD is to make webpages easily accessible on multiple platforms.

RWD – 3 core techniques

This next section will be about the three main core techniques with responsive web design. The three techniques are as listed.

  • Fluid layout
  • CSS media queries
  • Responsive media

Fluid layout- This is the process of using units such as px or em to resize elements to the way you want it to be displayed on your webpage. This can also be done using some other CSS aspects such as grids and flexbox.

CSS media queries - These are aspects that modify the viewport size (which we discussed at an early week) and the change in screen width due to the platform used. This usually uses styling elements as they have different capabilities for most devices.

Responsive media - This is the process of making images flexible on different platforms, altering the size of an element briefly so that it does not distort. This is especially useful when scaling down on web browsers.

CSS3 media queries

The focus of this section is media queries, this media type is implemented for features such as height, resolution and width. Media queries are able to match the media type of the device that the user is using.

There are simple methods when using media queries, this method is designed to target the viewport method. This type is the @media query, when placed within the CSS style sheet it checks whether the media type of the device is able to access the web page.

The use of the @media to target different media types enables for easy functionality on different devices. This is an approach which most web developers use as it eliminates the uses of two style sheets for different platforms. This can be done by using a combination of min and max values.

What is CSS?

Cascading styles sheets are essentially a declarative language that allows users to style their webpages, which will be seen on browsers. With the development of CSS this allows for easy styling on HTML pages. Listed to the right are a handful of styling options that can be achieved using CSS.

  • Border colour and size
  • Text colour and size
  • Background colour
  • Font style
  • Animations

When using CSS there are some important aspects to consider such as:

The structure of the of content within the HTML document must be validated for errors as these can have a negative effect on certain styles. Another consideration is that CSS style sheets is separated from your HTML document this means that all styling can be easily changed or modified. Introducing CSS can help the user experience and accessibility of webpages making text clearer or bolder and better for those who have disorders or impaired sight.

Selectors

Within this section we will be discussing selectors. This is the use of a tag when styling. Therefore, when styling the CSS must match the HTML element.

So, when styling < p > as a tag, the selector will be p. However, when stating a tag this might change the style of the tags you have already used. To overcome this issue, you can implement a class attribute which will only style the specific stated class. There are four different types of selectors that can be used to help styling, these selectors are…

  • Type selector
  • Class selector
  • ID selector
  • Pseudo-class selector

Normalize.css

A normalized.css style sheet is a CSS file that makes browsers render all elements to current standards. Essentially this is a small CSS file which helps provide a more efficient cross browser consistency in default styling of HTML.

The viewport meta element

This next section is about how to present your web pages on different formats. This can mean testing your web pages on multiple devices / browsers as the overall appearance will change the format and layout of your web page depending on what device it is being viewed on.

Within this section we will discuss how to resolve these issues as well as implement meta element which is beneficial when using your webpages on multiple platforms.

The new element we had learnt was the viewport element which is essentially scaling your web page to the width of the browser. This simply means that if someone was to access your website on other smaller devices such as a smart phone then your webpage should remain the same size. The reason we do this is because we want our webpages to behave the same on every device as this means user experience doesn’t get affected.

Below I will break down the meta element which looks like this < meta name="viewport" content="width=devicewidth, initial-scale=1"/> and explain what each part means and does.

  • "width=device-width" – This part of the meta simply states that the width of the viewport in pixels must be the same as the screen width of the device. This part is the key aspect that makes different devices act the same.
  • "initial-scale=1" – This end point of the meta is telling the browser to display the page when loading in full size. This replaces the scaling down feature.

Block and inline elements

This next topic of discussion was block and inline elements. Both elements are different kinds of tags, they can be broken down into these two key points shown on the right.

  • Block- Is any element that starts on a new line. An example of these are headings (<h1>), paragraphs (<p>) and lists (<ol>and<ul>).
  • Inline- These are elements that are found within blocks. An example of these are bold <b>.

When using block and inline elements there are some fundamental rules that need to be applied such as... block elements must contain a nested inline element as well as other block elements. Inline elements cannot contain block elements. This is due to the fact that block elements are considered a higher hatchery of tags compared to inline. This means that inline elements can only contain other inline elements and or content.

Well formed HTML = Elements nested correctly

When using HTML it is best to visualise elements as if they are contained within their own little box.
